DB 모델링
1. 엔티티 정의
  - 시스템에서 다룰 데이터 구조를 정의한다.
  1) 엔티티(entity; 테이블)를 찾아낸다.
  2) 속성(attribute)을 찾아낸다.
  3) 식별자(key)를 선정한다.
2. 정규화
  - 중복 데이터 제거한다.
  1) 제1정규화
    - 중복 컬럼이나 중복 데이터를 별도 테이블로 분리한다.
    - 부모/자식 관계를 맺는다.
  2) 제2정규화
    - 주키가 여러 개일 경우, 모든 컬럼은 모든 주키에 종속되어야 한다.
    - 만약 모든 주키에 종속되지 않는 컬럼이 있다면 별도 테이블로 분리한다.
    - 부모/자식 관계를 맺는다.
  3) 제3정규화 
    - 모든 컬럼은 주키에 종속되어야 한다.
    - 만약 주키에 종속되지 않고 다른 일반 컬럼에 종속되는 컬럼이 있다면 별도 테이블로 분리한다.
    - 부모/자식 관계를 맺는다.
3. 물리 모델링
  1) 도메인 정의 
    - 컬럼들을 쓰임새에 따라 여러 유형으로 정의한다.
    - 각 컬럼에 도메인을 부여한다.
  2) 검색 인덱스 정의
    - 조회에서 검색 조건으로 사용하는 컬럼은 인덱스로 선언한다.
    - 즉 별도의 색인 테이블을 생성한다. => 검색 속도 향상.
  3) 유니크 인덱스 정의 
    - 주키는 아니지만 중복되지 말아야할 컬럼을 선언한다.
  4) Not Null 여부 설정
    - 필수 입력 컬럼을 지정한다.
  5) 물리 모델명 정의
    - 논리 모델에서 사용한 이름을 물리 모델 이름으로 바꾼다.
    - 특정 DBMS에 맞추어 컬럼 이름을 정의한다. 
4. DBMS 적용
  1) 포워드 엔지니어링(forward engineering) 
    - 모델링 정보를 바탕으로 SQL문 생성 
  2) DBMS에 테이블 생성
  
 
* 식별자 선정 방법
- 한 개 이상의 컬럼을 묶어서 데이터를 구분하는 키로 사용한다.
- 만약 키로 사용할 적절한 컬럼이 없다면 임의의 컬럼을 만들어 사용한다.
  => 이때, 보통 일련번호 컬럼을 만든다.
- 테이블의 데이터를 구분하기 위해 지정한 컬럼들을 "주 키(Primary Key)"라고 부른다.
  
* 외부 키(Foreign Key)
- 부모 테이블에 있는 특정 데이터의 PK를 저장하는 컬럼.

* 관계
1) 비식별 관계
- FK 컬럼이 일반 컬럼으로 사용될 경우. 
- 관계 정보가 이 테이블에서 식별자로 사용되지 않는다.
- FK != PK

2) 식별 관계
- FK 컬럼이 PK로도 사용될 경우.
- 관계 정보가 이 테이블에서 식별자로 사용된다.
- FK == PK
